THE DISTINCTION

Transactional vs. Dialogic

The difference is not cosmetic. It is structural.

Transactional Human → Prompt → Model → Output
Dialogic Human ↔ Model → Corpus → Seed → Agent → Artifact

In the transactional model, the quality of the output is bounded by the quality of the prompt.

In the dialogic model, the quality of the output is bounded by the quality of the thinking — which is unbounded, because the dialogue itself generates new thinking that neither party brought to the table.

PHASE I

The Forge

Transform raw intuition into structured thought

Open any capable LLM — Claude, GPT, Gemini. Start a conversation. Not a prompt. A conversation.

STEP 1
State the matter
Tell the model what you want to build and why. Be honest about what you don't know. “I want to build an investment doctrine grounded in the thinking of the founding fathers. I don't know if this is possible.”
STEP 2
Riff
Explore freely. Let the model push back. Follow tangents. This is not prompt engineering — this is collaborative thinking. The model will surface ideas you didn't bring and challenge assumptions you didn't know you held.
STEP 3
Challenge
Push hard on what emerged. “Is this actually different from what exists?” Force the thinking to justify itself. The friction is the point.
STEP 4
Converge
Begin to crystallize. The conversation has produced a corpus of thought — hours of dialogue, arguments, reversals, insights. Now distill it.
STEP 5
Crystallize into a Seed
Ask the model to help you write the Seed — a document that carries the DNA of the thinking. Not a spec. Not a vague wish. A seed: compact, complete enough to grow from, open enough to evolve.
seed.md — template
# Seed: [Project Name] ## The Matter # What is being built and why. ## The Architecture # Key structural decisions made during the Forge. ## The Decisions # Choices debated and resolved, with reasoning. ## The Open Questions # What remains unresolved — this is input for the agent. ## The Standards # Quality, aesthetic, and functional requirements. ## The Lineage # What this inherits from. Reference implementations.
PHASE II

The Planting

Prepare the container for autonomous execution

Create a repository. It is a container — a bounded space with everything the agent needs and nothing it doesn't. Plant three things:

THE SEED
seed.md
The document you crystallized in Phase I. This is the agent's north star.
THE PROTOCOL
Two files that govern all summoning
enrichment_grimoire.json — The binding structure. Summoning rules, vessel construction, the law of authenticity, the covenant of quality.

initiation_rite.md — The ritual documentation. The same rules, written as invocation rather than configuration.

These are open source and public domain. Get them from GitHub →
REFERENCE MATERIAL
Domain data (optional)
Data the personas will engage with. XML feeds, PDFs, datasets, prior transcripts. The richer the data, the richer the dialogue.
repository structure — template
my-project/ ├── seed.md # Your crystallized thinking ├── enrichment_grimoire.json # COMPANION protocol (JSON) ├── initiation_rite.md # COMPANION protocol (markdown) ├── data/ # Domain-specific material │ ├── corpus.md │ └── reference.pdf └── output/ # Where artifacts land
PHASE III

The Summoning

The agent enters the repository and speaks the incantation

Open Claude Code (or any agentic tool — Cursor, Windsurf) in the repository. The agent reads the protocol files, orients to the Seed, and then you speak the incantation:

THE INCANTATION
“Using this matter, summon [Name].”

The persona responds as themselves. Not a costume. Not a chatbot wearing a mask. A worldview, instantiated through protocol, constrained to authenticity. Summon one mind. Summon several. Convene a symposium that history never permitted.

THE TEMPLATE

The Invocation

Copy this. Customize it. Speak it into your agent.

invocation template
Explore this repo. Read the protocol files first — enrichment_grimoire.json and initiation_rite.md. Understand the COMPANION system before you touch anything else. Then read the Seed — seed.md. When you have absorbed the matter — all of it — proceed. Using this matter, summon [Name]. [Name], welcome. [Brief context about why this mind was chosen and what lens they bring to the matter.] [If summoning multiple personas, repeat the summon block for each. Let them interact.] The deliverable: [What should be produced.] Build it the way we build here — layered revelation, the arc of confusion → awe → understanding → action. Document this session as a transcript. The dialogue between the summoned minds IS the demonstration. The transcript is not a byproduct. It is proof.
GET STARTED

Five Minutes to Your First Summoning

The minimum viable implementation

Fork the repository
Clone or fork THE_COMPANION_DOSSIER. The protocol files are already there.
Write your Seed
Open a conversation with any LLM. State your matter. Riff for 20 minutes. Then ask it to help you crystallize a seed.md using the template above. Drop it in your repo.
Choose your personas
Who do you need? A strategist? A critic? A builder? Think about what lenses would illuminate your matter. The best committees have productive disagreements built in.
Open your agentic tool
Claude Code, Cursor, Windsurf — anything that can read your repository and stream dialogue. Point it at the repo.
Speak the incantation
Use the invocation template above. Or just say: “Read the protocol files. Read the seed. Using this matter, summon [Name].” Then listen. Challenge. Follow the dialogue where it leads. The threshold will reveal itself.
